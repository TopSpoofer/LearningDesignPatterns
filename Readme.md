# OO 原则

1. 封装变化
2. 多用组合，少用继承
3. 针对接口编程，不针对实现编程
4. 为了交互对象之间的松耦合设计而努力
5. 类应该对扩展开放，对修改关闭
6. 依赖倒置原则，要依赖抽象，不要依赖具体类
7. 最少知识原则：只和你的密友谈话
8. 好莱坞原则：别调用我们，我们会调用你
